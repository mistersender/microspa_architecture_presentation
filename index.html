Let's Talk Vernacular

SPA

Single-Page Application

(Typically uses a JS framework)

Monolith

Single, large code base.

* a single large code base (or potentially 2-3 code bases for MVCs etc) that control all of the code for a site (or sites). * typically have a lot of shared components * once a monolith is learned, it's typically super fast to code in

Micro SPA

  • Code base broken out into separate, distinct pieces
  • Performs a single task
  • Intended to work as part of a whole website, but can stand alone.

MicroSPA vs Monolith Repos

Using a generic E-Commerce site

Monolith Repos:

  • widgets_galore

MicroSPA Repos:

  • widgets_galore_portal
  • home_page
  • sales_landing_page
  • search
  • checkout
  • my_account
  • update_user_info

can reuse my_account etc on other portals

portal is a wrapper for the site.

Why Choose MicroSPA over a Monolith?

it really comes down to one question...

Ask yourself, which is more valued:

Speed or Agility?

Speed

The strength of the Monolith

Tons of reusable featues

Learning curve can be high, but lowers quickly over time.

Can move fast because of repetition & replicability.

Agility

The strength of the MicroSPA

Quick, painless deploy process

Little to no coordination

No need for complex git methodologies

Merge conflicts rare

Site upgrades can be done in chunks (or swarmed)

Learning curve to start is lower

but requires agile engineers

Specialization

Easier to own areas

Don't have to agonize over framework decisions

Encourages Expiramentation

Low-impact to try something in one MicroSPA
